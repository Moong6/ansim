# 케어알림장 — 로그인 화면 전환 링크 추가 패치 v1.0

> **이 문서의 성격**: 1~6차 + UX 미세조정 패치가 **이미 적용된 상태**에 얹는 **초소형 UI 패치**입니다. 새 기능·DB·API 변경 모두 없음. 단일 컴포넌트의 추가/수정만.
>
> **패치 범위**: 직원 로그인 화면(`/login`)과 보호자 로그인 화면(`/login-parent`) 하단에 **상호 전환 링크** 추가. 5차의 두 페이지 분리 결정은 유지하되, 사용자가 잘못 들어왔을 때 한 번에 이동할 수 있도록.
>
> **변경 총량(매우 작음)**: 두 페이지(`pages/Login.tsx`, `pages/LoginParent.tsx`) 하단 영역만 수정. 백엔드 변경 없음.

## 1. 변경 사항

### 직원 로그인 화면 `/login` — 하단 추가
```
┌──────────────────────────────┐
│  케어알림장                  │
│                              │
│  이메일 [_________________]  │
│  비밀번호 [_______________]  │
│                              │
│  [   로그인   ]              │
│                              │
│  ─────────────────────────   │
│  보호자이신가요?  가족 로그인 →│  ← 신규
└──────────────────────────────┘
```

### 보호자 로그인 화면 `/login-parent` — 하단 추가
```
┌──────────────────────────────┐
│  케어알림장 · 가족 로그인    │
│                              │
│  이메일 [_________________]  │
│  비밀번호 [_______________]  │
│                              │
│  [   로그인   ]              │
│                              │
│  ─────────────────────────   │
│  직원이신가요?  직원 로그인 →│  ← 신규(또는 기존 텍스트 강화)
└──────────────────────────────┘
```

### 디자인 디테일

| 요소 | 내용 |
|---|---|
| 위치 | 로그인 카드 가장 하단, [로그인] 버튼 아래 충분한 여백 |
| 구분선 | 얇은 회색 가로선(`border-top` 1px 또는 hr) 위로 한 줄 |
| 안내 텍스트 | "보호자이신가요?" / "직원이신가요?" — 작은 폰트, 회색 |
| 링크 | "가족 로그인 →" / "직원 로그인 →" — 본문 색 또는 약한 강조, 화살표 `→` 또는 ` ›` |
| 클릭 영역 | 텍스트와 링크 전체가 클릭 영역 (모바일/태블릿 터치 용이성) |
| 동작 | React Router `<Link to="/login-parent">` 또는 `/login` — SPA 내 이동, 새로고침 없음 |

### 톤·시각 일관성
- 직원 화면의 링크는 직원 화면 톤(차분한 흰 배경에 어울리는 색)
- 보호자 화면의 링크는 보호자 화면 톤(미색 `#FBF7F2` 배경에 어울리는 부드러운 색)
- 5차 두 화면의 시각적 차이는 그대로 유지 — 링크 한 줄만 양쪽에 대칭으로 추가

## 2. 5차 정책과의 관계

- **두 페이지 분리 유지**: `/login`과 `/login-parent`는 여전히 별도 페이지. 한 페이지 안에서 탭으로 토글하지 않음.
- **로그인 후 라우팅 가드 그대로**: 5차에서 만든 "보호자 토큰으로 직원 경로 접근 시 `/parent/home` 리다이렉트, 반대도 마찬가지" 가드 변경 없음. 이 패치는 **로그인 전** 단계의 사용자 도움 장치일 뿐.
- **에러 메시지 정책 유지**: 5차에서 "직원 로그인 화면에서 보호자 계정으로 로그인 시도 시 안내 + 링크"라고 명시돼 있다면 그 메시지도 그대로. 이 패치의 하단 링크는 그것과는 별개로, **로그인 시도 전부터** 보이는 안내.

---

## 3. 접근성 (a11y) 짧은 메모

- 링크에 `aria-label` 명시 권장 (예: `aria-label="가족 로그인 페이지로 이동"`).
- 키보드 탭 순서: 이메일 → 비밀번호 → 로그인 버튼 → 하단 링크. 자연스러운 순서로.
- 링크 호버·포커스 상태에 색 변화 또는 밑줄 표시.
